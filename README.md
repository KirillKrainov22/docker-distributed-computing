# Лаба 4 — Контейнеризация с Docker

Распределённое вычисление простых чисел по схеме **Producer / Consumer** с координацией через Redis.

## Структура проекта

```
lab4/
├── producer/
│   ├── Dockerfile
│   └── app.py          # инициализирует задачу в Redis
├── consumer/
│   ├── Dockerfile
│   └── app.py          # берёт чанки и считает простые числа
├── docker-compose.yml
├── .env                # не в репозитории, см. ниже
└── README.md
```

## Как это работает

1. **Producer** читает параметры из `.env`, записывает их в Redis и завершается
2. **Consumer-ы** параллельно берут чанки через атомарную операцию `INCRBY`
3. Redis гарантирует что два контейнера никогда не получат один диапазон
4. Результаты каждого чанка пишутся в список `results` в Redis
5. Когда все чанки разобраны — каждый consumer выводит итоговое количество простых чисел

## Настройка

Создай файл `.env` в корне проекта:

```env
RANGE_START=0
RANGE_END=10000000
CHUNK_SIZE=500000
```

| Переменная | Описание |
|---|---|
| `RANGE_START` | Начало диапазона проверки |
| `RANGE_END` | Конец диапазона проверки |
| `CHUNK_SIZE` | Размер одного куска работы на одного воркера |

## Запуск

```bash
# Собрать образы и запустить с 1 воркером
docker compose up --build --scale consumer=1

# Запустить с несколькими воркерами
docker compose up --scale consumer=4

# Остановить всё
docker compose down
```

## Масштабирование вживую

Пока воркеры уже работают, в новом терминале:

```bash
docker compose up --scale consumer=8 --no-recreate
```

Флаг `--no-recreate` не трогает уже запущенные контейнеры — только добавляет новые. Новые воркеры сразу подключатся к Redis и начнут разбирать оставшиеся чанки.
